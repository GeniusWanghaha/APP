# 发布包策略

## 构建维度
- Flavor:
  - `demo`：演示环境默认配置
  - `contest`：竞赛提交配置
- BuildType:
  - `debug`
  - `release`

## 命名规则
输出 APK 自动命名：
- `PhotoSentinel-demo-debug-v<version>.apk`
- `PhotoSentinel-contest-debug-v<version>.apk`
- `PhotoSentinel-contest-release-v<version>.apk`

## 推荐命令
- 演示包：`./gradlew assembleDemoDebug`
- 竞赛调试包：`./gradlew assembleContestDebug`
- 竞赛发布包：`./gradlew assembleContestRelease`

## 交付建议
- 对外演示优先 `demoDebug`
- 评审提交使用 `contestRelease`
- 交付前执行 `lint + test + assemble`
- 目录瘦身策略：`dist/packages` 默认仅保留 `contestRelease`，其余 debug 包按需现构建
