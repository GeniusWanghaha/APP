# 硬件接入说明（ECG + PPG）

## 1. 已保留并稳定的手机端接口
- 领域模型：`app/src/main/java/com/photosentinel/health/domain/model/HardwareContracts.kt`
- 统一仓储接口：`app/src/main/java/com/photosentinel/health/domain/repository/HardwareInterfaceRepository.kt`
- 当前 BLE 实现：`app/src/main/java/com/photosentinel/health/data/repository/BleHardwareBridgeRepository.kt`

以上接口已经稳定，后续更换固件实现时，不需要改动上层 ViewModel 与 UI 调用方式。

## 2. 协议字段对齐（APP 当前使用）
- `FrameID` -> `HardwareFrame.frameId`
- `T_base_us` -> `HardwareFrame.baseTimestampMicros`
- `ecg_raw[]` -> `HardwareFrame.ecgSamples`
- `ppg_red[]` -> `HardwareFrame.ppgRedSamples`
- `ppg_ir[]` -> `HardwareFrame.ppgIrSamples`
- `state_flags` -> `HardwareFrame.stateFlags`
- `crc16` -> `HardwareFrame.crc`

## 3. 状态位映射表（用于 UI 提示）
- `bit0` ECG 导联异常 -> “请检查电极接触”
- `bit3` PPG FIFO 溢出 -> “存在采样拥塞，已触发门控”
- `bit4` PPG 中断超时 -> “采样节拍异常，请复测”
- `bit5` ADC 饱和 -> “信号削顶，建议减小按压或调整增益”
- `bit6` BLE 背压 -> “链路拥塞，已启用丢包保护”
- `bit7` 传感器就绪 -> `true` 时可输出高阶指标
- `bit8` 手指接触 -> `true` 时可输出 PPG 相关指标
- `bit9` 同步标记 -> 收到 `SYNC_MARK` 后在下一帧置位（单次事件）

## 4. 当前支持的控制命令（固件 + APP 协议层）
- `START`：开始采集并输出数据通知
- `STOP`：停止采集
- `SET_CFG`：支持 LED PA、`ppgPhaseUs`、`ppgLatencyUs`、温度开关、日志级别、PPG 模式
- `GET_INFO`：读取 control/status 快照
- `SELF_TEST`：通过 status 中 `self_test_pass_bitmap/self_test_fail_bitmap` 读取自检结果
- `SYNC_MARK`：注入同步事件标记，用于手机端时间轴/日志对齐

## 5. 可配置项（工程模式页面）
- 设备名前缀
- Service/Data/Control/Status UUID
- MTU
- PPG 相位补偿 `ppgPhaseUs`
- PPG 固定延时 `ppgLatencyUs`

配置保存在本地设置仓储中，重启 APP 仍会生效。
