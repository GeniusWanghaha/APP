# 算法逻辑总览（软硬件联动版）

更新时间：2026-04-02  
适用版本：`algo-v2.0`（手机端 Kotlin 实现）  
对应代码：
- `app/src/main/java/com/photosentinel/health/data/repository/ble/BlePacketProtocol.kt`
- `app/src/main/java/com/photosentinel/health/infrastructure/signal/TimelineReconstructor.kt`

## 1. 总体数据流

1. ESP32-S3 通过 BLE 发送分段数据包（含 ECG、PPG、状态位、微秒级基准时间戳）。
2. 手机端完成分包重组、协议校验、CRC 校验，解码为 `HardwareFrame`。
3. 基于 `baseTimestampMicros` 重建 ECG/PPG 时间轴，处理抖动和短时丢帧补点。
4. 对 ECG、PPG 做滤波和峰值检测，提取 HR、SpO2、PTT/PAT、HRV、形态学等指标。
5. 通过 SQI（信号质量）门控决定输出层级（基础/高阶/研究）。
6. 输出风险事件与趋势解释，供 UI 展示和报告生成。

---

## 2. BLE 协议与解包逻辑

## 2.1 分段包协议
- 协议版本：`0x01`
- 数据消息类型：`0xA1`
- 帧负载总长：`132 bytes`
- 分段头长度：`8 bytes`

头部检查流程：
1. 校验包长度 >= 8。
2. 校验 `protocolVersion`、`messageType`。
3. 校验 `segmentIndex < segmentCount`。
4. 校验 `payloadLength` 在合法范围内。

## 2.2 分段重组
使用 `FrameSegmentAssembler`：
- 按 `frameId` 聚合分段。
- 全部分段到齐后按 `segmentIndex` 排序拼接。
- 超过 `staleThresholdMillis=2000ms` 的不完整帧丢弃。

## 2.3 帧解码与 CRC
流程：
1. 校验负载长度必须等于 `132`。
2. 对前 `130` 字节执行 `CRC16-CCITT-FALSE`，与尾部 CRC 比较。
3. 解出字段：
   - `frameId: UInt16`
   - `baseTimestampMicros: Int64`
   - `ecgCount, ppgCount`
   - `stateFlags: UInt16`
   - ECG：最多 10 点（16bit）
   - PPG Red/IR：最多 16 点（24bit）

## 3. 时间轴重建（微秒级）

模块：`TimelineReconstructor`

## 3.1 采样周期计算
- `ecgPeriodUs = 1_000_000 / ecgSampleRateHz`
- `ppgPeriodUs = 1_000_000 / ppgSampleRateHz`
- 默认配置来自 `BleRuntimeConfig`，并通过 `sanitized()` 限幅。

## 3.2 基准时间戳稳定化
- 若当前 `baseTimestampMicros` 比上一帧理论最小时间还小（超出抖动容忍），则抬升到期望值。
- 抖动容忍：`JITTER_TOLERANCE_US = 3000`

## 3.3 PPG 相位与链路延迟
- PPG 基准时间 = `stableBaseTimestamp + ppgPhaseUs + ppgLatencyUs`
- `ppgPhaseUs/ppgLatencyUs` 优先取硬件状态包，缺失时回退到本地配置。

## 3.4 丢帧与补点
- 使用 frameId（16bit 回绕）计算丢失序列。
- 若丢帧数量 `<=2`（短缺口），做线性插值补点，仅用于显示连续性。
- `metricsUsable = droppedFrameIds.isEmpty()`：有丢帧时高阶指标降级。

## 4. ECG 处理链路（AD8232）

## 4.1 滤波
- 陷波：`50Hz`（`Q=12`）
- 高通：`0.5Hz`
- 低通：`40Hz`

## 4.2 Pan-Tompkins 风格特征
在带通后信号上进行：
1. 一阶导：`d[n] = x[n] - x[n-1]`
2. 平方：`s[n] = d[n]^2`
3. 移动积分窗口（MWI）：
   - 窗长：`ECG_MWI_WINDOW_SAMPLES = 38`
   - 最小有效点数：`ECG_MWI_MIN_SAMPLES = 12`

## 4.3 自适应双阈值 R 峰检测

峰候选条件：
- 积分特征局部极大值
- 满足不应期：`>= 260ms`

动态阈值公式：
- `threshold = noise + 0.25 * (signal - noise)`

双阈值策略：
- 主阈值：`integrated >= T_i && bandAbs >= T_b`
- 回搜阈值：主阈值的 `0.5`（仅在长时间未检出时允许）

漏检恢复：
- `rrMissTimeoutMicros = clamp(avgRR * 1.66, 700ms, 2400ms)`
- 若超过该时长仍无峰值，允许回搜阈值触发。

信号/噪声水平更新（IIR）：
- `newLevel = 0.875 * old + 0.125 * current`

## 5. PPG 处理链路（MAX30102）

## 5.1 预处理
- IR 通道先做 DC 估计（低通 `0.4Hz`），得到 AC 分量。
- AC 再经高通 `0.5Hz` + 低通 `8Hz`，用于足点/峰值检测。

## 5.2 足点与峰值检测

足点（局部最小）条件：
- 局部最小
- 不应期：`>=220ms`
- 斜率门限：`max(0.7, noiseStd*0.15)`

峰值（局部最大）条件：
- 必须有 pending foot
- 振幅门控：`pulseAmplitude = peak - foot`
  - `>= max(0.9, medianPulseAmplitude * 0.35)`
  - `<= 5000`
- 若 foot 超时 `>650ms` 仍无峰值，则丢弃该 foot。

## 5.3 时序指标
- `PTT = foot_ts - matched_rpeak_ts`（80~450ms 窗口匹配）
- `PAT = peak_ts - matched_rpeak_ts`
- `RiseTime = peak_ts - foot_ts`
- `HalfWidth ≈ RiseTime * 1.6`（当前为近似估计）

---

## 6. SpO2 算法（借鉴 MAX30102 开源思路）

## 6.1 窗口与质量前置
- 窗口：最近 `4s`
- 最小样本：`320`
- 红光/红外必须同步可用

## 6.2 信号整理
1. 分别减去均值（去 DC）。
2. 线性去趋势（最小二乘斜率 beta）。
3. 4 点移动平均（用于 valley 检测）。
4. IR 反相后做“峰值检测”得到 valley 索引。

## 6.3 分段 AC/DC 比值
对相邻 valley 区间：
1. 找 Red/IR 的区间峰值及峰值索引。
2. 用两端点做线性基线，计算 AC 分量（峰值减基线）。
3. 计算每搏比值：
   - `ratio = (Red_AC * IR_DC) / (IR_AC * Red_DC)`
4. 过滤合法区间：`0.2 ~ 1.84`
5. 全窗口取 ratio 中位数作为最终比值。

## 6.4 SpO2 映射
- `SpO2 = (-45.060 * ratio + 30.354) * ratio + 94.845`
- 输出夹紧到 `80~100%`

## 6.5 相关性门控
- 计算 detrend 后 Red/IR 皮尔逊相关系数。
- 若相关系数 `<0.45`，则本窗口 SpO2 无效（返回 null，保持平滑值）。

## 7. 指标计算逻辑

## 7.1 基础指标
- `HR = 60000 / mean(RR_ms)`，RR 合法区间 `320~2000ms`
- `PTT`：多搏平均
- `SpO2`：见第 6 章

## 7.2 HRV
- `SDNN`：RR 标准差
- `RMSSD`：相邻 RR 差分均方根
- `pNN50`：相邻 RR 差分 > 50ms 比例

## 7.3 PPG 形态与灌注
- `PI = (std(red) / mean(red)) * 100`
- `ReflectionIndex = clamp((std/mean)*120, 0.15, 0.85)`

## 7.4 联合一致性与节律
- `BeatPulseConsistency = 1 - |CV(RR) - CV(PTT)| * 4`
- `ArrhythmiaIndex = (SDNN / RRmean) * 100`

## 7.5 血管弹性与血管年龄（模型指标）
- `PWV = 0.45m / (PTT_s)`，范围限制 `3~15 m/s`
- `ElasticityScore`：
  - `100 - (PWV-6)*12 - |HR-72|*0.25 + SQI*10`
- `VascularAge`：
  - `23 + (PWV-5)*8 + (HR>90 ? 2 : 0)`

---

## 8. SQI 与输出门控

## 8.1 SQI 组成
1. 硬件状态位惩罚（导联脱落、手指未检测、FIFO 溢出、BLE 背压等）。
2. ECG 振幅+能量因子。
3. PPG 振幅因子。
4. Red/IR 相关性因子。
5. ratio 合法性惩罚。
6. RR-PTT 一致性加权。

SQI 最终范围：`0.12 ~ 1.0`

## 8.2 输出层级
- `BASELINE`：`!metricsUsable` 或 `SQI < 0.65`
- `ADVANCED`：`0.65 <= SQI < 0.82`
- `RESEARCH`：`SQI >= 0.82`

---

## 9. 风险事件规则（当前版本）

- 心动过速：`HR >= 110`
- 心动过缓：`HR <= 52`
- 节律不齐提示：`ArrhythmiaIndex >= 11`
- 房颤风险初筛：`ArrhythmiaIndex >= 15` 且 RR 可用
- 疑似早搏提示：`BeatPulseConsistency < 0.45`

每条事件都带 `confidence`，按 SQI 加权。

---

## 10. 趋势文本规则（血压相关）

- `PWTT < 120ms`：偏高风险趋势
- `PWTT > 220ms`：偏低风险趋势
- 其余：个体参考区间

说明：仅做趋势参考，不是医疗诊断值。

## 11. 关键参数总表（当前实现）

### 11.1 采样与滤波
- ECG 采样率：`250Hz`
- PPG 采样率：`400Hz`
- ECG 陷波：`50Hz`

- ECG 低通：`40Hz`
- PPG DC 低通：`0.4Hz`
- PPG 高通：`0.5Hz`

### 11.2 时序阈值
- R 峰不应期：`260ms`
- PPG 足点不应期：`220ms`
- PTT 匹配窗：`80~450ms`
- PPG foot 超时：`650ms`

### 11.3 SpO2
- 窗口：`4s`
- 最小样本：`320`
- 相关性下限：`0.45`
- ratio 合法区间：`0.2~1.84`

### 11.4 SQI 与门控
- 输出降级阈值：`0.65`
- 研究级阈值：`0.82`

## 12. 文档实时更新规则（后续必须执行）

从现在开始，任何算法改动都必须在**同一次提交**同步更新本文件，至少更新以下 4 项：

1. 改动点：具体公式/阈值/流程变更了什么。
2. 影响面：影响哪些指标（HR/SpO2/PTT/SQI/风险事件等）。
3. 兼容性：是否影响历史数据可比性、是否要升级 `algorithmVersion`。
4. 验证证据：至少附 1 条命令与结果（如 `testDemoDebugUnitTest`）。

建议同步维护：
- `HomeViewModel` 中 `algorithmVersion` 字段；
- `MeasurementSession.algorithmVersion`；
- 测试用例中的阈值断言。
