# 手机端采集与输出策略（v2026-04-03）

## 1. 当前策略
- 连接硬件后，APP 进入 **60 秒采集阶段**。
- 采集阶段仅显示：
  - 会话状态
  - 采集进度（秒数 + 进度条）
  - 链路状态与调试摘要
- 采集阶段 **不实时展示测量指标**。
- 采集满 60 秒后，一次性发布本轮结果：
  - 核心指标
  - 高阶指标（若质量门控通过）
  - 波形展示
  - 风险提示与 AI 文本摘要

## 2. 质量门控与异常处理
- 若达到 60 秒但当帧质量不足，系统不会输出伪结果，而是继续采集并提示用户调整接触状态。
- 仅当本轮确实发布过结果时，才允许写入历史记录，避免未完成采集污染历史库。

## 3. 关键代码位置
- 60 秒采集窗口与发布门控：  
  `app/src/main/java/com/photosentinel/health/ui/viewmodel/HomeViewModel.kt`
- 首页交互与进度展示：  
  `app/src/main/java/com/photosentinel/health/ui/screens/HomeScreen.kt`

## 4. 与离线验证工程的关系
- `project_root` 的 Python 主要用于公开数据集验证与报告产出。
- `android_project` 的 Kotlin 负责真机实时链路。
- 共享夹具的一致性对齐结果见：  
  `project_root/outputs/reports/kotlin_python_alignment_status.md`
